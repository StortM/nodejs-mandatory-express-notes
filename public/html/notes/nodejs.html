node

The node command is the entry point to interacting with node.js runtime environment.

When using this command without any arguments, in a terminal, it is possible to write and execute Javascript code directly in the terminal.

Alternatively it is possible to run a Node.js script (a Javascript file) from the command line. This is done with the node <path to .js file> command.

npm

npm (node package manager) is a package manager, like Maven for Java, but for the Javascript language. It is the default package manager for Node.js and consists of 3 parts:

- The npm website, which can be used to discover packages and more.
- The CLI, which is used to run npm commands from a terminal.
- The registry, which is a large public database of Javascript packages. This is where packages are downloaded from when running the npm install command.

package.json

The package.json is a json file that all npm packages contains, usually in the root directory.
The file holds metadata relevant to the project.
This could be the project name, version, needed dependencies and so on.

Following the JSON data format the fields are divided into key:value pairs,
with the possibility for objects, like the dependencies object, to hold multiple key:value pairs.

Example of this projects package.json
{
  "name": "mandatory-1-doc-website",
  "description": "A website built with Node.js and express"
     "detailing what has been learned so far in the Node.js elective",
  "author": "Mads Kjærgaard Christensen <mads38g6@stud.kea.dk>",
  "version": "1.0.0",

  "dependencies": {
    "express": "^4.17.1"
  },

  "scripts": {
    "start": "node app.js",
    "start-dev": "set PORT=3000 & nodemon app.js"
  }
}

npm init

The npm init command can be used to create a package.json file from scratch, if no such file exists.

When running the command with no arguments, a short guide appears asking you questions to help populate the resulting package.json.
As an example you will be asked to name the package, provide a version (1.0.0 by default), provide an url to the git repository (if any) etc.

npm install

This command is used to install packages (or dependencies) in a Node project.

It can be used with no arguments, where npm will find every dependency listed in the package.json file, and install them by adding them to the node_modules folder.

A package name can also be provided as an argument, which will cause npm to look up the package at npmjs.com and, if found, add the package to the node_modules folder and list it as a dependency in the package.json file.

No arguments
npm install

With package name
npm install <package name>

npm run

This command will run an arbitrary command from the package.json file.

Commands can be specified in the "scripts" object, which can contain multiple commands

If no command name is specified as an argument, a list of available commands (if any) will be returned.

No arguments
npm run

Command name specified
npm run <command>

An example of the usage of the run command could be through a package.json file with 2 scripts defined.

An ordinary start script, which will run the app.js file normally through node, and a developer focused start-dev script, which will attempt to create and set an environment variable PORT to 3000, and run the app.js file through nodemon.

Example of a scripts object
"scripts": {
    "start": "node app.js",
    "start-dev": "set PORT=3000 & nodemon app.js"
}

nodemon

"Nodemon is a tool that helps develop node.js based applications by automatically restarting the node application when file changes in the directory are detected."

This makes it a very useful development tool as any files changes will cause nodemon to re-run your application, making your changes available without the hassle of stopping the running node process and starting it again.

Usage

To run a javascript file through nodemon you replace the normal the node command with nodemon:

nodemon <js file>
instead of
node <js file>

Installation

Nodemon is recommended to be installed globally, but can also be installed as a development dependency.

If installed globally it will be available to be run by every Node.js project on the system. If installed locally as a development dependency, it will only be available to the speific project in which it was installed.

Global install
npm install -g nodemon

Local install as development dependency
npm install --save-dev nodemon
